# FXP
- Числа с фиксированной точкой
- На дробную часть выделяется фиксированное количество бит (m бит), которые образуют двоичное число.
- Если каждому биту слева направо последовательно присвоить индекс от 1 (включительно) до m (включительно), то при переводе в десятичную систему (классическим способом) получим число A. При переводе в дробную часть получим число B. Интересно, что A = m * B.
- Любое десятичное число можно представить в двоичном коде, поэтому дробную часть B нужно умножить на m, чтобы получившееся число перевести в двоичное – оно и будет последовательностью битов дробной части.
- Если дробная часть B кратна числу 1 / 2^m, то число A будет целым (и мы его легко переведём в двоичный код), но если B не кратна 1 / 2^m, то число A будет нецелым. В таком случае A округляем до ближайшего целого и уже его переводим в двоичный код. Этот процесс называется квантованием. Именно в этот момент и появляется шум.
1.	Если FLP считать за достаточно точное представление и сравнить с ним получившийся результат в FXP, то каждое число будет иметь свой шум: noise[i]=flp[i]-fxp[i]. В худшем случае шум одного числа равен 1 / 2^(m+1). Это происходит, если число FLP оказывается ровно по середине интервала между точно представимыми значениями B.
2.	При сложении чисел, шум складывается, а конкретно равен abs(sum).
3.	При умножении шум перемножается, а конкретно равен abs(multiplication). Пока шум квантования обоих множителей <= 1, операция умножения никогда не увеличивает шум. Однако если шум > 1, то операция умножения увеличивает шум по экспоненте (такой случай возможен только в промежуточных вычислениях, но не на входе, т. к. даже 0 бит на дробную часть обеспечивают точность <= 1).
4.	Из (1) и (2) следует, что если многократно складывать числа с шумом 1 / 2^(m+1), то за 2^(m+1) таких операций шум достигнет 1, а дальше с помощью умножения легко можно получить экспоненциально большой шум.
5.	Пусть D – количество операций сложения, за которые в худшем случае можно превысить шум T. Тогда D = T / (1/2^(m+1)) = T * 2^(m+1).
6.	Если по ТЗ требуется шум не более 10^-9, то при следующем количестве бит m его можно превысить за следующее количество операций:
- m=16; D = T * 2^(m+1) = 10^-9 * 2^17 = 0,000131072 (значит точность 10^-9 недостижима даже для одного числа без единой операции, что легко проверить)
- m=29; D = 10^-9 * 2^30 = 1,073741824 (всего за 2 операции шум может превысить 10^-9)
- m=64; D = 10^-9 * 2^65 = 3,7 * 10^10 (мало для современных процессоров)
- m=96; D = 10^-9 * 2^97 = 1,6 * 10^20 (вполне достаточно, чтоб за 100 лет не накопилась ошибка больше 10^-9 даже в худшем случае)
